# Synchronizer

Synchronizer держит каталоги в согласованном состоянии между десктопом на Linux
и телефоном на Android по локальной сети. Он написан, чтобы переносить хранилища
Obsidian - обычные папки с Markdown-файлами - в обе стороны без облачного
аккаунта, без сервера синхронизации в интернете и без гадания, "какая копия
свежее". Обе стороны используют один и тот же движок синхронизации и
согласовывают изменения до того, как что-либо будет записано на диск.

## Как это выглядит для пользователя

Устройства находят друг друга в одной сети и показываются списком - только те,
что доступны прямо сейчас. Вы выбираете одно и подтверждаете сопряжение на обоих
экранах: там показывается одинаковый шестизначный код, и пока вы не убедитесь,
что он совпадает, ничего не происходит. Сопряжение симметрично, начать может
любая сторона.

Дальше вы отмечаете каталоги, которыми это устройство делится (например, два
хранилища Obsidian из нескольких, остальные не трогаются), а на втором
устройстве при первой синхронизации выбираете папку, куда они лягут. Спрашивается
это один раз.

Синхронизация двусторонняя: один тап на каталоге. Приложение сканирует обе
копии, сравнивает их с тем, на чём они сошлись в прошлый раз, и раскладывает
изменения по смыслу: что придёт сюда, что уйдёт на другое устройство, что
слилось само и что требует решения.

## Слияние

Правки не выбираются целыми файлами. Если вы дописали строку в конец заметки, а
на телефоне поправили заголовок, обе правки останутся - файл сливается построчно,
как это делает система контроля версий, и никто ни о чём не спрашивает.

Спрашивается только там, где обе стороны переписали одни и те же строки
по-разному. Такое место называется спорным, и решается оно по отдельности, а не
файлом целиком: рядом с каждым спорным местом видно обе версии и две кнопки -
взять своё или взять чужое. Как только вы решили, это место уходит с экрана, и
остаётся ровно то, что ещё ждёт вас. Что не тронули - остаётся в вашей версии.
Всё остальное в файле сливается само.

Перед применением показывается полный предпросмотр: какие файлы и папки
создадутся и удалятся, а внутри каждого файла - построчная разница с номерами
строк, удалённое красным с минусом, добавленное зелёным с плюсом. Длинные куски
нетронутого текста сворачиваются с пометкой, сколько строк свёрнуто. Ни один байт
не записывается, пока вы не нажмёте "Применить".

Есть и простой режим (переключатель в настройках): изменения применяются без
предпросмотра, что сливается - сливается само, остальное решается в пользу
стороны, изменённой позже.

## Почему сделано именно так

Сравнение идёт по содержимому файлов, а не по времени изменения. Каждый файл
сводится к хешу SHA-256; два файла с одинаковым хешем - это один и тот же файл
независимо от того, когда их последний раз трогали, что избавляет от бесконечного
перекопирования, в которое скатываются инструменты, смотрящие только на
timestamp. Снимок каталога - набор файлов и их хешей - называется манифестом.

Одних хешей мало: они говорят, что файл изменился, но не говорят, каким он был.
Поэтому рядом с каталогом лежит `.synchronizer/base` - копии файлов на момент
последней синхронизации. Это и есть общий предок, без которого построчное слияние
невозможно, а любое расхождение выглядело бы конфликтом. Каталог начинается с
точки, поэтому не попадает в манифесты и не синхронизирует сам себя - тот же
приём, на котором стоит `.git`.

Обнаружение сделано на UDP-мультикасте, а не на mDNS/Avahi: это чистый Dart, без
нативных плагинов и без зависимости от системного демона. Сессия
аутентифицируется общим секретом, выведенным при сопряжении (HMAC-SHA256,
взаимно: обе стороны доказывают, что знают секрет). Переданное содержимое
сверяется с хешем на обоих концах, поэтому битые байты не попадают в хранилище.
Запись всегда атомарная - во временный файл и переименование.

Логика синхронизации написана один раз и общая для обоих приложений, поэтому
десктоп и телефон в принципе не могут разойтись в том, что считать изменением.

## Структура репозитория

```
linux/            Flutter-приложение для десктопа на Linux
android/          Flutter-приложение для Android
shared/sync_core/ Движок на чистом Dart: обход каталогов, хеширование,
                  манифесты, построчный diff, трёхстороннее слияние
shared/sync_net/  Сеть на чистом Dart: обнаружение, сопряжение, сессия,
                  сведение каталогов и предпросмотр
shared/sync_ui/   Общий интерфейс на Flutter, одинаковый на обеих платформах
LICENSE           GNU General Public License, версия 2
```

Ядро и сеть не зависят от Flutter, поэтому их можно тестировать и использовать
отдельно. Поддержка Windows возможна как более позднее дополнение и сейчас
намеренно вне области работ.

## Сборка

Нужен Flutter SDK (3.44 или новее) с Dart 3.12 или новее.

Десктоп:

```
cd linux
flutter run -d linux
```

Телефон (при подключённом устройстве с включённой отладкой по USB):

```
cd android
flutter run -d android
```

Готовый APK:

```
cd android
flutter build apk --release
```

Тесты:

```
cd shared/sync_core && flutter test
cd shared/sync_net && flutter test
cd shared/sync_ui  && flutter test
```

## Состояние

Работает и используется по назначению. Сделаны обнаружение устройств,
сопряжение с подтверждением кода, транспорт, двусторонняя синхронизация,
построчное слияние с предпросмотром и разрешением по спорным местам, журнал
синхронизаций, темы и цветовые схемы, автозапуск, русский и английский язык.

Что стоит знать заранее:

- Трафик не шифруется. Сессия аутентифицируется общим секретом, и чужое
  устройство не получит ваши файлы, но сам обмен идёт открытым TCP. Это домашняя
  сеть, а не интернет; TLS - в планах.
- Файл передаётся целиком и целиком помещается в память, ограничение - 64 МиБ на
  файл. Для заметок это несущественно, для больших вложений - да.
- Первая синхронизация папки, которая уже есть на обоих устройствах, сливает по
  общим строкам: предка ещё нет, и отличить "здесь добавили" от "там удалили"
  невозможно. Ошибка в этом случае делается в сторону сохранения текста, и
  приложение об этом честно предупреждает. Со второй синхронизации слияние
  точное.
- На Android приложение просит доступ ко всем файлам - иначе нельзя писать в
  папку Obsidian, выбранную вами. Это личная сборка; в Google Play с таким
  разрешением не пускают.
- Слияние построчное, поэтому применимо к тексту. Двоичные файлы (картинки,
  вложения) синхронизируются целиком, а их конфликты решаются выбором стороны.
