# Synchronizer

Synchronizer держит каталоги в согласованном состоянии между десктопом на Linux
и телефоном на Android по локальной сети. Он написан, чтобы переносить хранилища
Obsidian - обычные папки с Markdown-файлами - в обе стороны без облачного
аккаунта, без сервера синхронизации в интернете и без гадания, "какая копия
свежее". Обе стороны используют один и тот же движок синхронизации и
согласовывают изменения до того, как что-либо будет записано на диск.

## Как это выглядит для пользователя

Десктоп находит телефоны в той же сети и показывает их списком. Вы выбираете один
и подтверждаете сопряжение на самом телефоне, так что соединение никогда не
устанавливается втихаря. После сопряжения десктоп предлагает каталоги, которыми
он может делиться; вы выбираете нужные (например, два хранилища Obsidian из
нескольких, а остальные не трогаете). На телефоне вы указываете, куда сохранять
каждый входящий каталог.

Дальше синхронизацию может начать любая из сторон. Каждое устройство сканирует
свою копию каталога, стороны обмениваются результатами, и вычисляется разница:
какие файлы добавлены, у каких изменилось содержимое, какие удалены.

Применить эту разницу можно двумя способами, выбор в настройках:

- Интерактивный. Приложение показывает, что именно изменится, и ждёт вашего
  подтверждения, прежде чем тронуть хоть один файл, - та же идея, что и просмотр
  предлагаемой правки перед её принятием.
- Простой. Разница применяется напрямую, без запроса по каждому изменению. Удобно,
  когда конкретной паре устройств вы уже доверяете и просто хотите, чтобы папки
  совпали.

## Почему сделано именно так

Сравнение идёт по содержимому файлов, а не по времени изменения. Каждый файл
сводится к хешу SHA-256; два файла с одинаковым хешем - это один и тот же файл
независимо от того, когда их последний раз трогали, что избавляет от бесконечного
перекопирования, в которое скатываются инструменты, смотрящие только на
timestamp. Снимок каталога - набор файлов и их хешей - называется манифестом, и
синхронизация по сути сводится к тому, чтобы взять разницу двух манифестов и
привести их в соответствие. Термины (снимок, diff, применение) намеренно взяты из
систем контроля версий, потому что эта модель совпадает с тем, чего люди и так
ожидают от работы с файлами на разных машинах.

Логика синхронизации написана один раз и общая для обоих приложений, поэтому
десктоп и телефон в принципе не могут разойтись в том, что считать изменением.

## Структура репозитория

```
linux/            Flutter-приложение для десктопа на Linux
android/          Flutter-приложение для Android
shared/sync_core/ Движок на чистом Dart, общий для обоих приложений: обход
                  каталогов, хеширование содержимого, манифесты и их сравнение
LICENSE           GNU General Public License, версия 2
```

Поддержка Windows возможна как более позднее дополнение и сейчас намеренно вне
области работ.

## Сборка

Нужен Flutter SDK (3.44 или новее) с Dart 3.12 или новее.

Десктоп:

```
cd linux
flutter run -d linux
```

Телефон (при подключённом устройстве с включённой отладкой по USB):

```
cd android
flutter run -d android
```

У общего движка есть свои тесты:

```
cd shared/sync_core
dart test
```

## Состояние

Ранняя стадия. Общий движок умеет сканировать каталог в манифест и вычислять
разницу двух манифестов в виде набора изменений, готового к просмотру; эта часть
покрыта тестами. Обнаружение устройств, сопряжение, транспорт между устройствами
и два режима применения ещё предстоит сделать.
